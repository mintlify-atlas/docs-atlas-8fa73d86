## Overview

`ReactiveLineEdit` is a reactive wrapper around Qt's `QLineEdit` that provides two-way data binding with your Pydux store. User input automatically updates the store, and store changes automatically update the input field.

**Source:** `ppg_runtime/ReactiveWidgets/line_edit.py:16`

## Inheritance

```python
ReactiveLineEdit(QLineEdit)
```

Inherits all methods and properties from Qt's `QLineEdit`.

## Constructor

```python
ReactiveLineEdit(
    parent,
    key: str,
    placeholder: str = "",
    onChange = None,
    **kwargs
)
```

### Parameters

<ParamField path="parent" type="QWidget" required>
  Parent widget containing the Pydux store. Must have `subscribe_to_store()` and `update_store()` methods.
</ParamField>

<ParamField path="key" type="str" required>
  Store key to bind to. Supports nested keys using dot notation (e.g., `"user.name"`).
  
  This is the key that will be read from and written to in the store.
</ParamField>

<ParamField path="placeholder" type="str" default="">
  Placeholder text displayed when the input is empty.
  
  Example: `"Enter your name"`
</ParamField>

<ParamField path="onChange" type="Callable[[str], None]" default="None">
  Optional callback function called when the text changes.
  
  Receives the new text value as argument. Called on user input, not on programmatic updates from store.
</ParamField>

<ParamField path="**kwargs" type="dict">
  Additional keyword arguments passed to `QLineEdit` constructor.
</ParamField>

## Methods

### _update_from_store()

```python
def _update_from_store(self)
```

Internal method that synchronizes the input field with the store value. Called automatically when store changes.

**Behavior:**
- Retrieves value from store using `parent.get_nested(key)`
- Updates text field only if value differs (prevents cursor jumps)
- Uses `_updating_from_store` flag to prevent circular updates
- Supports nested keys

**Source:** `ppg_runtime/ReactiveWidgets/line_edit.py:66`

### on_store_change(store)

```python
def on_store_change(self, store)
```

Callback invoked automatically when the Pydux store changes. Delegates to `_update_from_store()`.

<ParamField path="store" type="dict">
  The updated store state.
</ParamField>

**Source:** `ppg_runtime/ReactiveWidgets/line_edit.py:79`

### _on_text_changed(text)

```python
def _on_text_changed(self, text: str)
```

Internal callback connected to Qt's `textChanged` signal. Updates the store when user types.

**Behavior:**
- Checks `_updating_from_store` flag to prevent circular updates
- For nested keys, uses `parent.update_nested_model()`
- For simple keys, uses `parent.update_store()`
- Triggers `onChange` callback if provided

<ParamField path="text" type="str">
  The new text value entered by the user.
</ParamField>

**Source:** `ppg_runtime/ReactiveWidgets/line_edit.py:47`

## Usage Examples

### Basic Text Input

Simple single-line text input:

```python
from ppg_runtime.ReactiveWidgets import ReactiveLineEdit
from PySide6.QtWidgets import QMainWindow
from ppg_runtime.application_context import PPGLifeCycle, Pydux

class MyWindow(QMainWindow, PPGLifeCycle, Pydux):
    def component_will_mount(self):
        self.set_schema({"username": str})
        self.subscribe_to_store(self)
    
    def render_(self):
        ReactiveLineEdit(
            self,
            key="username",
            placeholder="Enter your username"
        ).setGeometry(20, 20, 200, 30)
```

### Nested State

Access nested objects in the store:

```python
class MyWindow(QMainWindow, PPGLifeCycle, Pydux):
    def component_will_mount(self):
        self.set_schema({
            "user": {
                "name": str,
                "email": str,
                "phone": str,
            }
        })
        self.subscribe_to_store(self)
    
    def render_(self):
        # Multiple fields editing nested state
        ReactiveLineEdit(self, key="user.name", placeholder="Name").setGeometry(20, 20, 200, 30)
        ReactiveLineEdit(self, key="user.email", placeholder="Email").setGeometry(20, 60, 200, 30)
        ReactiveLineEdit(self, key="user.phone", placeholder="Phone").setGeometry(20, 100, 200, 30)
```

### With Change Callback

React to user input:

def on_username_change(text):

    # Validate input

## Important Notes

- **Two-way binding**: Changes flow both ways automatically
- **Circular update prevention**: Uses `_updating_from_store` flag to prevent infinite loops
- **Nested key support**: Use dot notation for nested objects (e.g., `"user.profile.name"`)
- **Cursor position**: Preserved during programmatic updates
- **All QLineEdit methods work**: setMaxLength(), setValidator(), setInputMask(), etc.
