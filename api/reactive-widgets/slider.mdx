## Overview

`ReactiveSlider` is a reactive wrapper around Qt's `QSlider` that provides two-way data binding with your Pydux store. User interactions automatically update integer values in the store, and store changes automatically update the slider position.

**Source:** `ppg_runtime/ReactiveWidgets/slider.py:20`

## Inheritance

```python
ReactiveSlider(QSlider)
```

Inherits all methods and properties from Qt's `QSlider`.

## Constructor

```python
ReactiveSlider(
```

### Parameters

<ParamField path="parent" type="QWidget" required>
  Parent widget containing the Pydux store. Must have `subscribe_to_store()`, `update_store()`, and `get_nested()` methods.
</ParamField>

<ParamField path="key" type="str" required>
  Store key to bind to. Supports nested keys using dot notation (e.g., `"settings.volume"`).

</ParamField>

<ParamField path="orientation" type="Qt.Orientation" default="Qt.Horizontal">

  Import from `PySide6.QtCore` or equivalent.
</ParamField>

<ParamField path="onChange" type="Callable[[int], None]" default="None">
  Optional callback function called when the slider value changes.
  
  Receives the new integer value.

<ParamField path="minimum" type="int" default="0">

</ParamField>

<ParamField path="maximum" type="int" default="100">

## Methods

### _update_from_store()

```python
def _update_from_store(self)
```

Internal method that synchronizes the slider value with the store. Called automatically when store changes.

**Behavior:**
- Retrieves integer value from store using `parent.get_nested(key)`
- Defaults to `minimum` if key not found
- Converts value to integer
- Updates slider only if value differs
- Uses `_updating_from_store` flag to prevent circular updates
- Supports nested keys

**Source:** `ppg_runtime/ReactiveWidgets/slider.py:73`

### on_store_change(store)

```python
def on_store_change(self, store)
```

Callback invoked automatically when the Pydux store changes. Delegates to `_update_from_store()`.

<ParamField path="store" type="dict">
  The updated store state.
</ParamField>

**Source:** `ppg_runtime/ReactiveWidgets/slider.py:89`

### _on_value_changed(value)

```python
def _on_value_changed(self, value: int)
```

Internal callback connected to Qt's `valueChanged` signal. Updates the store when user moves the slider.

**Behavior:**
- Checks `_updating_from_store` flag to prevent circular updates
- For nested keys, uses `parent.update_nested_model()`
- For simple keys, uses `parent.update_store()`
- Triggers `onChange` callback if provided

<ParamField path="value" type="int">
  The new slider value.
</ParamField>

**Source:** `ppg_runtime/ReactiveWidgets/slider.py:56`

## Usage Examples

### Basic Horizontal Slider

Simple volume control:

```python
from ppg_runtime.ReactiveWidgets import ReactiveSlider
from PySide6.QtWidgets import QMainWindow
from ppg_runtime.application_context import PPGLifeCycle, Pydux

class MyWindow(QMainWindow, PPGLifeCycle, Pydux):
    def component_will_mount(self):
        self.set_schema({"volume": int})
        self.subscribe_to_store(self)
    
    def render_(self):
        ReactiveSlider(
            self,
            key="volume",
            minimum=0,
            maximum=100
        ).setGeometry(20, 20, 300, 30)
```

### With Live Value Display

Show current value with label:

```python
from ppg_runtime.ReactiveWidgets import ReactiveSlider, ReactiveLabel

class MyWindow(QMainWindow, PPGLifeCycle, Pydux):
    def component_will_mount(self):
        self.set_schema({
            "brightness": int,
        })
        self.update_store({"brightness": 50})  # Default value
        self.subscribe_to_store(self)
    
    def render_(self):
        # Slider
        ReactiveSlider(
            self,
            key="brightness",
            minimum=0,
            maximum=100
        ).setGeometry(20, 20, 300, 30)
        
        # Live value display
        ReactiveLabel(
            self,
            text="Brightness: {brightness}%"
        ).setGeometry(330, 20, 150, 30)
```

### Nested State

Access nested settings:

```python
class MyWindow(QMainWindow, PPGLifeCycle, Pydux):

            "audio": {

    def render_(self):
```


### Multi-Slider Control Panel

Multiple sliders for different settings:

```python
class MyWindow(QMainWindow, PPGLifeCycle, Pydux):
    def component_will_mount(self):
        self.set_schema({
            "red": int,
            "green": int,
            "blue": int,
        })
        self.update_store({"red": 128, "green": 128, "blue": 128})
        self.subscribe_to_store(self)
    
    def render_(self):
        # RGB color sliders
        y = 20
        for color in ["red", "green", "blue"]:
            ReactiveLabel(self, text=f"{color.capitalize()}: {{{color}}}").setGeometry(20, y, 100, 30)
            ReactiveSlider(
                self,
                key=color,
                minimum=0,
                maximum=255
            ).setGeometry(130, y, 300, 30)
            y += 40
```

## Important Notes

- **Integer values only**: Slider values are always integers
- **Default value**: Defaults to `minimum` if key not found in store
- **Two-way binding**: Changes flow both ways automatically
- **Circular update prevention**: Uses `_updating_from_store` flag
- **Nested key support**: Use dot notation for nested objects
- **All QSlider methods work**: setTickPosition(), setTickInterval(), setSingleStep(), etc.
- **Range validation**: Values automatically clamped to min/max range

## Float Values

For floating-point values, use a multiplier:

```python
def on_slider_change(value):
    float_value = value / 100.0  # Convert 0-100 to 0.0-1.0
    print(f"Float value: {float_value}")

ReactiveSlider(
    self,
    key="intValue",
    minimum=0,
    maximum=100,
    onChange=on_slider_change
)
```
