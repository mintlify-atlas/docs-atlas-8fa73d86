## Overview

`ReactiveSpinBox` is a reactive wrapper around Qt's `QSpinBox` that provides two-way data binding with your Pydux store. It allows users to enter or adjust integer values using keyboard input or increment/decrement buttons.

**Source:** `ppg_runtime/ReactiveWidgets/spin_box.py:16`

## Inheritance

```python
ReactiveSpinBox(QSpinBox)
```

Inherits all methods and properties from Qt's `QSpinBox`.

## Methods

### _update_from_store()

```python
def _update_from_store(self)
```

Internal method that synchronizes the spinbox value with the store. Called automatically when store changes.

**Behavior:**
- Retrieves integer value from store using `parent.get_nested(key)`
- Defaults to `minimum` if key not found
- Converts value to integer
- Updates spinbox only if value differs
- Uses `_updating_from_store` flag to prevent circular updates
- Supports nested keys

**Source:** `ppg_runtime/ReactiveWidgets/spin_box.py:67`

### on_store_change(store)

```python
def on_store_change(self, store)
```

Callback invoked automatically when the Pydux store changes. Delegates to `_update_from_store()`.

<ParamField path="store" type="dict">
  The updated store state.
</ParamField>

**Source:** `ppg_runtime/ReactiveWidgets/spin_box.py:83`

### _on_value_changed(value)

```python
def _on_value_changed(self, value: int)
```

Internal callback connected to Qt's `valueChanged` signal. Updates the store when user changes the value.

**Behavior:**
- Checks `_updating_from_store` flag to prevent circular updates
- For nested keys, uses `parent.update_nested_model()`
- For simple keys, uses `parent.update_store()`
- Triggers `onChange` callback if provided

<ParamField path="value" type="int">
  The new spinbox value.
</ParamField>

**Source:** `ppg_runtime/ReactiveWidgets/spin_box.py:50`

## Usage Examples

### Basic SpinBox

Simple integer input:

```python
from ppg_runtime.ReactiveWidgets import ReactiveSpinBox
from PySide6.QtWidgets import QMainWindow
from ppg_runtime.application_context import PPGLifeCycle, Pydux

class MyWindow(QMainWindow, PPGLifeCycle, Pydux):
    def component_will_mount(self):
        self.set_schema({"count": int})
        self.subscribe_to_store(self)
    
    def render_(self):
        ReactiveSpinBox(
            self,
            key="count",
            minimum=0,
            maximum=100
        ).setGeometry(20, 20, 100, 30)
```

class MyWindow(QMainWindow, PPGLifeCycle, Pydux):

        self.update_store({"quantity": 1})
        self.subscribe_to_store(self)

        ).setGeometry(110, 20, 100, 30)

### Form with Multiple SpinBoxes

Product order form:

```python
class MyWindow(QMainWindow, PPGLifeCycle, Pydux):
    def component_will_mount(self):
        self.set_schema({
            "order": {
                "shirts": int,
                "pants": int,
                "shoes": int,
            },
            "totalItems": int,
        })
        self.subscribe_to_store(self)
    
    def calculate_total(self, value):
        total = (
            self.store.get("order", {}).get("shirts", 0) +
            self.store.get("order", {}).get("pants", 0) +
            self.store.get("order", {}).get("shoes", 0)
        )
        self.update_store({"totalItems": total})
    
    def render_(self):
        y = 20
        items = [("shirts", "Shirts"), ("pants", "Pants"), ("shoes", "Shoes")]
        
        for key, label in items:
            ReactiveLabel(self, text=label).setGeometry(20, y, 100, 30)
            ReactiveSpinBox(
                self,
                key=f"order.{key}",
                minimum=0,
                maximum=10,
                onChange=self.calculate_total
            ).setGeometry(130, y, 80, 30)
            y += 40
        
        # Total
        ReactiveLabel(
            self,
            text="Total Items: {totalItems}"
        ).setGeometry(20, y, 200, 30)
```

## Important Notes

- **Integer values only**: SpinBox values are always integers (use QDoubleSpinBox for floats)
- **Default value**: Defaults to `minimum` if key not found in store
- **Two-way binding**: Changes flow both ways automatically
- **Circular update prevention**: Uses `_updating_from_store` flag
- **Nested key support**: Use dot notation for nested objects
- **All QSpinBox methods work**: setSingleStep(), setPrefix(), setSuffix(), etc.
- **Range validation**: Values automatically clamped to min/max range
- **Keyboard input**: Users can type values directly (validated on focus loss)
