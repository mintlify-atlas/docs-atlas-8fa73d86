## Overview

The `ppg init` command creates a new PPG project in your current directory. It guides you through an interactive setup process to configure your application's basic settings.

```bash
ppg init
```

<Warning>
This command will abort if a `src/` directory already exists in the current location to prevent overwriting existing projects.
</Warning>

## Interactive Setup

When you run `ppg init`, you'll be prompted for the following information:

### 1. App Name

Your application's display name. PPG automatically converts this to CamelCase format.

```bash
? App name: my awesome app
# Converted to: MyAwesomeApp
```

<Info>
**Name Conversion**: The `to_camel_case()` function handles snake_case, kebab-case, and space-separated names:
- `my_app` → `MyApp`
- `my-app` → `MyApp`  
- `my awesome app` → `MyAwesomeApp`
</Info>

### 2. Version

Your app's initial version number (default: `1.0.0`).

```bash
? Version: 1.0.0
```

### 3. Author

Author name (defaults to your system username).

```bash
? Author: John Doe
```

### 4. Qt Binding

Select which Qt Python binding to use:

- **PySide6** (recommended, default)
- PySide2
- PyQt6
- PyQt5

```bash
? Select your Qt binding [PyQt5/PyQt6/PySide2/PySide6] (PySide6):
> PySide6
```

<Check>
PPG automatically checks if the selected binding is installed. If not, it installs it using pip.
</Check>

### 5. Mac Bundle Identifier

For macOS apps, specify a unique bundle identifier (e.g., `com.author.appname`).

```bash
? Mac bundle identifier (e.g. com.john.myawesomeapp, optional): com.john.myawesomeapp
```

## Project Structure Created

After confirmation, `ppg init` creates the following structure:

```
project-root/
├── src/
│   ├── build/
│   │   └── settings/
│   │       ├── base.json       # Core settings
│   │       ├── mac.json        # macOS-specific settings
│   │       ├── linux.json      # Linux-specific settings
│   │       └── windows.json    # Windows-specific settings
│   └── main/
│       ├── python/
│       │   └── main.py         # Application entry point
│       ├── icons/
│       │   └── Icon.ico        # App icons
│       └── resources/          # Additional resources
```

## Generated Configuration

The `src/build/settings/base.json` file is populated with your choices:

```json
{
  "app_name": "MyAwesomeApp",
  "author": "John Doe",
  "version": "1.0.0",
  "binding": "PySide6",
  "main_module": "src/main/python/main.py",
  "hidden_imports": [
    "__future__"
  ]
}
```


## Error Handling

### Project Already Exists

```bash
$ ppg init

💔 Error: The src/ directory already exists.
Aborting.
```

### Installation Failure

If the Qt binding installation fails:

```bash
❌ Failed to install PySide6. Please check your internet connection and permissions, and try again.
```

## Implementation Details

The `init` command (from `builtin_commands/__init__.py:55-162`):

1. **Validates** that `src/` doesn't exist
2. **Prompts** for configuration values
3. **Displays** a confirmation table  
4. **Checks** if the Qt binding is installed
5. **Installs** the binding if missing
6. **Creates** the `src/` directory structure
7. **Copies** template files with variable substitution
8. **Generates** `base.json` with your settings
