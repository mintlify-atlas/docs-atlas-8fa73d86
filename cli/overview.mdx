## Introduction

The PPG CLI provides a comprehensive set of commands to manage your desktop application's entire lifecycle - from project initialization to building production-ready installers. All commands are invoked using `ppg <command>` in your terminal.

## Additional Commands

### ppg version

Show the installed PPG version:

```bash
ppg version
```

<Info>
  This command displays the current PPG version from `package.json`.
</Info>

### ppg sign

Sign your frozen application (platform-specific):

```bash
ppg sign
```

<Note>
  Currently only implemented for Windows. Signs all binary files in the freeze directory so the OS trusts your application. See [Deployment](/advanced/deployment#code-signing) for configuration.
</Note>

### ppg sign_installer

Sign your installer package:

```bash
ppg sign_installer
```

<Note>
  Signs the installer file created by `ppg installer`. Platform-specific implementation for Windows, Linux (GPG), and macOS.
</Note>

### ppg repo

Generate repository files for automatic updates:

```bash
ppg repo
```

<Info>
  Creates repository metadata files for Linux package managers (APT, Pacman, DNF/YUM). Used for setting up automatic update channels.
</Info>

### ppg upload

Upload your application to ppg.sh hosting:

```bash
ppg upload
```

<Warning>
  Requires ppg.sh account credentials. Uploads the installer and repository files to PPG's hosting service.
</Warning>

### ppg release

Manage release versions:

```bash
ppg release [version]
```

<Info>
  Updates version numbers, creates release builds, and optionally publishes to repositories. If version is omitted, prompts interactively.
</Info>

## Common Workflows

### Starting a New Project

```bash
# Create a new project
ppg init

# Run it in development mode
ppg start
```

### Development Cycle

```bash
# Create new components as you build
ppg create component

# Test your changes
ppg test

# Run the app to see changes
ppg start
```

### Building for Production

```bash
# Clean previous builds
ppg clean

# Compile to executable
ppg freeze

# Create installer
ppg installer
```

## Command Line Parser

PPG's CLI is built on Python's `argparse` module with custom command registration through the `@command` decorator. Each command is automatically discovered and made available when you run `ppg`.

### Getting Help

You can get help for any command:

```bash
# General help
ppg --help

# Command-specific help
ppg freeze --help
```

## Command Categories

### Project Management
- `init` - Initialize new projects
- `version` - Check PPG version

### Development
- `start` - Run from source
- `create` - Generate code scaffolding
- `test` - Run tests

### Build & Distribution
- `freeze` - Compile executable
- `installer` - Create installers
- `sign` - Sign application binaries
- `sign_installer` - Sign installer packages
- `clean` - Clean build artifacts

### Release & Publishing
- `repo` - Generate repository files
- `upload` - Upload to ppg.sh
- `release` - Manage releases
