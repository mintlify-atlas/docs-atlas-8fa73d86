## Overview

This example demonstrates how to create a hybrid application that combines native Qt widgets with web-based UIs. It shows how to embed a web view, establish bidirectional communication between Python and JavaScript using QWebChannel, and create seamless integration between native and web components.

## What This Example Demonstrates

- Embedding web content with `QWebEngineView`
- Python ↔ JavaScript communication using `BridgeManager`
- QWebChannel integration
- Event emission from Python to JavaScript
- Calling Python functions from JavaScript
- Native Qt widgets alongside web views
- Developer tools integration

## Complete Source Code

```python
import sys
from ppg_runtime.application_context.PySide6 import ApplicationContext
from ppg_runtime.application_context import PPGLifeCycle, Pydux, init_lifecycle, BridgeManager
from ppg_runtime.application_context.devtools.reloader import hot_reloading
from ppg_runtime.application_context.utils import app_is_frozen
from PySide6.QtWidgets import QMainWindow, QPushButton
from PySide6.QtCore import QUrl
from PySide6.QtWebEngineWidgets import QWebEngineView
from PySide6.QtWebEngineCore import QWebEnginePage

# --------------------------------------------------------------------------------------
# Important! Production Considerations for Hot Reloading
# --------------------------------------------------------------------------------------
# Hot reloading is a development tool that allows you to instantly see UI changes
# when you save a file. It's extremely useful for rapid prototyping and designing
# interfaces.
#
# However, this functionality is not designed for use in production environments.
# For the final version of your application, it is highly recommended to remove
# the code related to hot reloading, such as the `@hot_reloading` decorator
# and the `window._init_hot_reload_system(__file__)` call.
#
# Keeping hot reloading active in production can negatively impact the application's
# performance, stability, and security.
# --------------------------------------------------------------------------------------

@init_lifecycle
@hot_reloading
class Pwa(QMainWindow, PPGLifeCycle, Pydux):
    def component_will_mount(self):
        self.subscribe_to_store(self)

    def render_(self):

        # loads the embedded browser engine (WebEngineView) where the PWA will be displayed (index.html)
        self.engine = QWebEngineView(self)
        self.engine.load(QUrl.fromLocalFile(self.get_resource("index.html")))


        # Create a bridge for Python ↔ JavaScript communication using QWebChannel
        # You can create multiple bridges if needed by instantiating BridgeManager with different names
        self.bridge = BridgeManager(self.engine, "bridge")

        # Register an event called "callback"
        # When triggered from JS, it will call handle_callback() in Python
        self.bridge.register("callback", self.handle_callback)


        # Add buttons to demonstrate functionality (e.g., opening DevTools, emitting events to JS)
        self.button = QPushButton("Open DevTools", self, clicked=self.open_dev_tools)
        self.button2 = QPushButton("test emit", self, clicked=lambda: self.bridge.emit("callback", {"data": "Hello from Python!"}))

        self.setCentralWidget(self.engine)

    def handle_callback(self, payload):
        """This is a callback function that handles messages from JavaScript via the bridge.

        Args:
            payload (dict): The payload data sent from JavaScript.

        Returns:
            str: A response message indicating the result of the callback handling.
        """

        print("Received message from JS:", payload)

        # You can send back a response to JavaScript if needed by returning a value
        return f"USER: {payload['username']} logged in successfully! (handled by Python)"

    def open_dev_tools(self):
        """Opens the DevTools window for the embedded web engine."""

        self.dev_tools = QWebEngineView(self)
        if not self.engine.page().devToolsPage():
            self.engine.page().setDevToolsPage(QWebEnginePage(self.engine.page().profile(), self.dev_tools))
        self.dev_tools.setPage(self.engine.page().devToolsPage())
        self.dev_tools.setWindowTitle("DevTools")
        self.dev_tools.show()
        self.dev_tools.resize(500, self.height())
        self.dev_tools.move(self.width() - 500, 0)


        # Emit an event to JavaScript indicating that DevTools has been opened (for demonstration purposes)
        self.bridge.emit("devtools_opened", {"message": "DevTools opened"})

    def responsive_UI(self):
        self.setMinimumSize(640, 480)
        self.button2.move(100, 0)


if __name__ == '__main__':
    appctxt = ApplicationContext()
    window = Pwa()
    if not app_is_frozen():
        window._init_hot_reload_system(__file__)
    window.show()
    exec_func = getattr(appctxt.app, 'exec', appctxt.app.exec_)
    sys.exit(exec_func())
```

## Code Breakdown

### Setting Up the Web View

```python
# loads the embedded browser engine (WebEngineView) where the PWA will be displayed (index.html)
self.engine = QWebEngineView(self)
self.engine.load(QUrl.fromLocalFile(self.get_resource("index.html")))
```

- `QWebEngineView`: Embeds a Chromium-based browser
- `QUrl.fromLocalFile()`: Loads a local HTML file
- `get_resource()`: PPG helper to locate bundled resources

### Creating the Python-JavaScript Bridge

```python
# Create a bridge for Python ↔ JavaScript communication using QWebChannel
# You can create multiple bridges if needed by instantiating BridgeManager with different names
self.bridge = BridgeManager(self.engine, "bridge")
```

`BridgeManager` wraps QWebChannel for easier communication:
- First argument: The `QWebEngineView` instance
- Second argument: Bridge name (used in JavaScript to access it)
- You can create multiple bridges for different purposes

### Registering Python Callbacks

```python
# Register an event called "callback"
# When triggered from JS, it will call handle_callback() in Python
self.bridge.register("callback", self.handle_callback)
```

This makes the Python function callable from JavaScript.

### Handling Messages from JavaScript

```python
def handle_callback(self, payload):
    """This is a callback function that handles messages from JavaScript via the bridge.

    Args:
        payload (dict): The payload data sent from JavaScript.

    Returns:
        str: A response message indicating the result of the callback handling.
    """

    print("Received message from JS:", payload)

    # You can send back a response to JavaScript if needed by returning a value
    return f"USER: {payload['username']} logged in successfully! (handled by Python)"
```

Key points:
- Receives data from JavaScript as `payload`
- Can perform Python operations (database queries, file I/O, etc.)
- Returns values back to JavaScript

### Emitting Events to JavaScript

```python
self.button2 = QPushButton("test emit", self, 
                          clicked=lambda: self.bridge.emit("callback", {"data": "Hello from Python!"}))
```

`bridge.emit()` sends events from Python to JavaScript:
- First argument: Event name
- Second argument: Payload data (must be JSON-serializable)

## JavaScript Side: Calling Python

In your `index.html` or JavaScript file:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Hybrid App</title>
    <script src="qrc:///qtwebchannel/qwebchannel.js"></script>
</head>
<body>
    <h1>Hybrid App Example</h1>
    <button onclick="loginUser()">Login</button>
    <div id="message"></div>

    <script>
        let bridge;

        // Initialize QWebChannel
        new QWebChannel(qt.webChannelTransport, function(channel) {
            // Access the bridge by the name you gave it ("bridge")
            bridge = channel.objects.bridge;

            // Listen for events from Python
            bridge.callback.connect(function(payload) {
                console.log("Received from Python:", payload);
                document.getElementById('message').innerText = payload.data;
            });

            bridge.devtools_opened.connect(function(payload) {
                console.log("DevTools status:", payload.message);
            });
        });

        // Call Python function from JavaScript
        function loginUser() {
            if (bridge) {
                // Call the registered "callback" function in Python
                bridge.invoke("callback", {username: "john_doe"}, function(response) {
                    // This runs when Python returns a value
                    console.log("Python response:", response);
                    document.getElementById('message').innerText = response;
                });
            }
        }
    </script>
</body>
</html>
```

## Communication Flow

### From JavaScript to Python

```
┌─────────────────────────────────────────────────────────────┐
│ 1. JavaScript: bridge.invoke("callback", {data: ...})       │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. QWebChannel transmits message                             │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. Python: handle_callback(payload) is called                │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. Python processes and returns result                       │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. JavaScript: callback function receives response           │
└─────────────────────────────────────────────────────────────┘
```

### From Python to JavaScript

```
┌─────────────────────────────────────────────────────────────┐
│ 1. Python: self.bridge.emit("event_name", {data: ...})      │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. QWebChannel transmits event                               │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. JavaScript: bridge.event_name.connect() handler called    │
└─────────────────────────────────────────────────────────────┘
```

## Use Cases for Hybrid Apps

### When to Use Hybrid Architecture

1. **Web-based UI with native features**: Use web technologies (HTML/CSS/JS) for UI while accessing native features through Python
2. **Existing web app**: Wrap an existing web application in a native shell
3. **Rich text and complex layouts**: HTML/CSS is often easier for complex layouts than Qt widgets
4. **Cross-platform UI**: Share HTML/CSS/JS across platforms while having platform-specific Python code
5. **Web API integration**: Easier to use modern web APIs and JavaScript libraries

### Example Use Cases

- **Dashboard apps**: Web-based charts and graphs with Python data processing
- **Chat applications**: Web UI for messages, Python for encryption and networking
- **Media players**: HTML5 video/audio with Python for file management
- **Development tools**: Web-based code editor with Python for compilation/execution

## BridgeManager API Reference

### Creating a Bridge

```python
bridge = BridgeManager(web_engine_view, "bridge_name")
```

### Registering Python Functions

```python
# Simple function
bridge.register("my_function", self.my_handler)

# Multiple functions
bridge.register("login", self.handle_login)
bridge.register("logout", self.handle_logout)
bridge.register("fetch_data", self.fetch_data)
```

### Emitting Events to JavaScript

```python
# Emit with data
bridge.emit("user_logged_in", {"username": "john", "token": "abc123"})

# Emit without data
bridge.emit("app_ready", {})
```

### Handler Function Signature

```python
def my_handler(self, payload):
    # payload is a dict from JavaScript
    print(payload)
    
    # Optionally return data back to JavaScript
    return {"success": True, "message": "Processed"}
```

## Multiple Bridges

You can create multiple bridges for different purposes:

```python
# Authentication bridge
self.auth_bridge = BridgeManager(self.engine, "auth")
self.auth_bridge.register("login", self.handle_login)
self.auth_bridge.register("logout", self.handle_logout)

# Data bridge
self.data_bridge = BridgeManager(self.engine, "data")
self.data_bridge.register("fetch", self.fetch_data)
self.data_bridge.register("save", self.save_data)
```

In JavaScript:

```javascript
let auth, data;

new QWebChannel(qt.webChannelTransport, function(channel) {
    auth = channel.objects.auth;
    data = channel.objects.data;
    
    auth.invoke("login", {username: "john"});
    data.invoke("fetch", {id: 123});
});
```
