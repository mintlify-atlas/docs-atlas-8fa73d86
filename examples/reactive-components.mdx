## Overview

This example demonstrates PPG's reactive widget system, which automatically synchronizes UI components with the Pydux store. When the store state changes, reactive widgets update automatically without manual intervention.

## What This Example Demonstrates

- Using reactive widgets (`Label`, `TextEdit`)
- Automatic UI updates when store state changes
- Two-way data binding between UI and store
- Schema definition for type safety
- Store subscription and reactive keys

## Complete Source Code

```python
import sys
from ppg_runtime.application_context.PySide6 import ApplicationContext
from ppg_runtime.application_context import PPGLifeCycle, Pydux, init_lifecycle
from ppg_runtime.application_context.devtools.reloader import hot_reloading
from ppg_runtime.application_context.utils import app_is_frozen
from ppg_runtime.ReactiveWidgets import Label, TextEdit
from PySide6.QtCore import Qt
from PySide6.QtWidgets import QMainWindow

# --------------------------------------------------------------------------------------
# Important! Production Considerations for Hot Reloading
# --------------------------------------------------------------------------------------
# Hot reloading is a development tool that allows you to instantly see UI changes
# when you save a file. It's extremely useful for rapid prototyping and designing
# interfaces.
#
# However, this functionality is not designed for use in production environments.
# For the final version of your application, it is highly recommended to remove
# the code related to hot reloading, such as the `@hot_reloading` decorator
# and the `window._init_hot_reload_system(__file__)` call.
#
# Keeping hot reloading active in production can negatively impact the application's
# performance, stability, and security.
# --------------------------------------------------------------------------------------

@init_lifecycle
@hot_reloading
class Reactive(QMainWindow, PPGLifeCycle, Pydux):
    def component_will_mount(self):
        self.subscribe_to_store(self)

        # Define the schema for the store
        self.set_schema({
            "notes": str,
        })

    def render_(self):
        """
            This is an example of using Reactive Widgets with Pydux store,
            the widgets will automatically update when the store changes using the key provided
        """
        Label(self, key="notes", alignment=Qt.AlignCenter).move(20, 20)
        TextEdit(self, key="notes", placeholder="Type your notes here...").setGeometry(20, 120, 600, 300)


    def responsive_UI(self):
        self.setMinimumSize(640, 480)


if __name__ == '__main__':
    appctxt = ApplicationContext()
    window = Reactive()
    if not app_is_frozen():
        window._init_hot_reload_system(__file__)
    window.show()
    exec_func = getattr(appctxt.app, 'exec', appctxt.app.exec_)
    sys.exit(exec_func())
```

## Code Breakdown

### Schema Definition

```python
def component_will_mount(self):
    self.subscribe_to_store(self)
    
    # Define the schema for the store
    self.set_schema({
        "notes": str,
    })
```

Before rendering, we define the store schema. This ensures type safety and defines what data the store will hold.

### Reactive Widgets

```python
def render_(self):
    Label(self, key="notes", alignment=Qt.AlignCenter).move(20, 20)
    TextEdit(self, key="notes", placeholder="Type your notes here...").setGeometry(20, 120, 600, 300)
```

The magic happens with the `key` parameter:

- **`Label(self, key="notes")`**: Creates a label that automatically displays the value of `store["notes"]`
- **`TextEdit(self, key="notes")`**: Creates a text editor that:
  - Displays the current value of `store["notes"]`
  - Automatically updates the store when the user types
  - Updates itself when the store changes from elsewhere

### How Reactivity Works

1. **User types in the TextEdit widget**
   - The widget automatically calls `update_store({"notes": new_text})`
   - The store state changes

2. **Store change triggers update**
   - All reactive widgets with `key="notes"` are notified
   - The Label automatically updates to show the new text
   - The TextEdit stays in sync

3. **No manual `on_store_change` needed**
   - Unlike the [Pydux Store example](/examples/pydux-store), you don't need to manually update widgets
   - Reactive widgets handle everything automatically

## Reactive vs Manual Updates

### With Reactive Widgets (This Example)

```python
# Automatic - just provide a key
Label(self, key="notes")
TextEdit(self, key="notes")
```

### Without Reactive Widgets (Manual)

```python
# Manual - you handle updates yourself
self.label = QLabel(self)
self.text_edit = QTextEdit(self)

def on_store_change(self, _):
    self.label.setText(self.store.get("notes", ""))
```
