## What is PPG?

PPG (Python Package Generator) is a modern and powerful framework for building scalable desktop applications with Python and Qt. It brings the best practices from web development—particularly React and Redux—to the world of native desktop applications.

<Note>
  PPG is built on top of Qt (PyQt5/6 or PySide2/6), combining the power of native widgets with modern development patterns.
</Note>

## Why Choose PPG?

Traditional Qt development can become complex and hard to maintain as applications grow. PPG solves this by providing:

- **Structured Component Lifecycle**: Clear hooks for every stage of a component's life
- **Type-Safe State Management**: Predictable state updates with Pydux
- **Developer Productivity**: Hot reloading for instant feedback
- **Flexible Architecture**: Support for native widgets, web UIs, or both

## Component-Based Architecture

PPG introduces a component-based architecture to Qt development. Instead of managing a monolithic application, you build composable components with well-defined lifecycles:

```python
from ppg_runtime.application_context import PPGLifeCycle, init_lifecycle
from PySide6.QtWidgets import QWidget

@init_lifecycle
class MyComponent(QWidget, PPGLifeCycle):
    def component_will_mount(self):
        # Setup before rendering
        pass
    
    def render_(self):
        # Build your UI
        pass
    
    def component_did_mount(self):
        # React to completed render
        pass
```

This lifecycle approach provides clear moments to:

- **Mount**: Set up components and fetch initial data
- **Update**: React to data changes and re-render the UI
- **Unmount**: Clean up resources to prevent memory leaks

## State Management with Pydux

Pydux is PPG's global state management system, inspired by Redux but built for Python with Pydantic integration:

```python
from ppg_runtime.application_context import Pydux
from pydantic import BaseModel

class UserModel(BaseModel):
    name: str
    email: str

class MyApp(Pydux):
    def __init__(self):
        # Define schema
        self.set_schema({
            'user': UserModel,
            'count': int
        })
        
        # Update state
        self.update_store({
            'count': 0,
            'user': UserModel(name='John', email='john@example.com')
        })
```

**Benefits:**
- **Type Safety**: Pydantic ensures your state is always valid
- **Unified State**: Single source of truth for your entire application
- **Reactive Updates**: Components automatically update when relevant state changes

<Warning>
  **Deprecation Notice:** `PPGStore` has been deprecated in favor of Pydux. Migrate to Pydux to leverage type safety and advanced features.
</Warning>

## Hybrid Development

PPG supports hybrid applications that combine native Qt widgets with modern web UIs:

- **Native Widgets**: Use Qt widgets for performance-critical parts
- **Web UIs**: Embed React, Vue, or HTML/CSS/JS interfaces
- **Bidirectional Communication**: `WebEngineBridge` provides seamless Python ↔ JavaScript messaging
- **Shared State**: Both native and web components can use the same Pydux state

## Comparison with Traditional Qt

| Feature | Traditional Qt | PPG |
|---------|---------------|-----|
| **Architecture** | Event-driven, no structured lifecycle | Component-based with lifecycle hooks |
| **State Management** | Manual signal/slot wiring | Global state with Pydux |
| **Development Speed** | Full restarts for changes | Hot reloading |
| **Scalability** | Can become complex at scale | Modular, maintainable components |
| **Type Safety** | Limited | Full Pydantic integration |

## Complete CLI Workflow

PPG includes a comprehensive CLI that manages your entire development lifecycle:

```bash
ppg init        # Initialize a new project
ppg start       # Run in development mode
ppg create      # Generate components/views
ppg freeze      # Compile to executable
ppg installer   # Create platform installer
```
