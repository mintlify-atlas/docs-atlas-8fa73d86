## Overview

This guide will walk you through creating your first PPG application from scratch. You'll learn the essential workflow: initializing a project, understanding the structure, running the app, and making your first changes with hot reloading.

<Note>
  **Prerequisites**: Make sure you have [installed PPG](/installation) before proceeding.
</Note>

## Create Your First Project

Let's build a simple "Hello World" application using PPG's interactive project generator.

<Steps>
  <Step title="Initialize the Project">
    Navigate to an empty directory and run:
    
    ```bash
    ppg init
    ```
    
    PPG will greet you with an interactive setup wizard:
    
    ```
    ✨ Welcome to PPG v1.2.0 ✨
    
    Let's create a new project! This will create a src/ directory with the necessary files and folders.
    ```
  </Step>
  
  <Step title="Configure Your Project">
    Answer the prompts to configure your project:
    
    ```bash
    App name: MyApp
    Version: 1.0.0
    Author: John Doe
    Select your Qt binding [PyQt5/PyQt6/PySide2/PySide6]: PySide6
    Mac bundle identifier: com.johndoe.myapp
    ```
    
    <Info>
      PPG automatically installs your chosen Qt binding if it's not already present.
    </Info>
  </Step>
  
  <Step title="Confirm Settings">
    Review your configuration and confirm:
    
    ```
    ┌─────────────────────────────────────┐
    │  Please confirm your settings       │
    ├─────────────────────────────────────┤
    │ App name:              MyApp        │
    │ Version:               1.0.0        │
    │ Author:                John Doe     │
    │ Qt binding:            PySide6      │
    │ Mac bundle identifier: com.johndoe… │
    └─────────────────────────────────────┘
    
    Continue? [y/n]: y
    ```
  </Step>
  
  <Step title="Project Created">
    PPG creates your project structure:
    
    ```
    🔎 Checking if PySide6 is installed...
    ✅ PySide6 is installed!
    
    🎉 Created the src/ directory 🎉. If you have PySide6 installed, you can now do:
    
        ppg start
    ```
  </Step>
</Steps>

## Project Structure

PPG generates a complete project structure for you:

```
my-app/
├── src/
│   ├── build/
│   │   └── settings/
│   │       ├── base.json       # App configuration
│   │       ├── mac.json         # macOS-specific settings
│   │       ├── linux.json       # Linux-specific settings
│   │       └── windows.json     # Windows-specific settings
│   └── main/
│       ├── python/
│       │   └── main.py          # Your application entry point
│       ├── resources/
│       │   └── base/            # Static resources (images, etc.)
│       └── icons/
│           ├── Icon.ico         # Windows icon
│           ├── base/            # Cross-platform icons
│           ├── linux/           # Linux icons
│           └── mac/             # macOS icons
```

## Understanding the Generated Code

Let's examine the generated `main.py` file:

```python main.py
import sys
from ppg_runtime.application_context.PySide6 import ApplicationContext
from ppg_runtime.application_context import PPGLifeCycle, Pydux, init_lifecycle
from ppg_runtime.application_context.devtools.reloader import hot_reloading
from ppg_runtime.application_context.utils import app_is_frozen
from PySide6.QtWidgets import QMainWindow, QLabel

@init_lifecycle
@hot_reloading
class MyApp(QMainWindow, PPGLifeCycle, Pydux):
    def component_will_mount(self):
        # Called before rendering - set up state and subscriptions
        self.subscribe_to_store(self)

    def render_(self):
        # Build your UI here
        QLabel('Hello World!', parent=self)

    def responsive_UI(self):
        # Handle responsive layout and sizing
        self.setMinimumSize(640, 480)


if __name__ == '__main__':
    appctxt = ApplicationContext()
    window = MyApp()
    if not app_is_frozen():
        window._init_hot_reload_system(__file__)
    window.show()
    exec_func = getattr(appctxt.app, 'exec', appctxt.app.exec_)
    sys.exit(exec_func())
```

### Code Breakdown

<Accordion title="Decorators">
  ```python
  @init_lifecycle
  @hot_reloading
  class MyApp(QMainWindow, PPGLifeCycle, Pydux):
  ```
  
  - `@init_lifecycle`: Automatically invokes lifecycle methods in the correct order
  - `@hot_reloading`: Enables hot reloading for instant UI updates during development
</Accordion>

<Accordion title="Multiple Inheritance">
  ```python
  class MyApp(QMainWindow, PPGLifeCycle, Pydux):
  ```
  
  - `QMainWindow`: Base Qt window class
  - `PPGLifeCycle`: Provides React-style lifecycle hooks
  - `Pydux`: Enables global state management
</Accordion>

<Accordion title="Lifecycle Methods">
  ```python
  def component_will_mount(self):
      self.subscribe_to_store(self)
  
  def render_(self):
      QLabel('Hello World!', parent=self)
  
  def responsive_UI(self):
      self.setMinimumSize(640, 480)
  ```
  
  - `component_will_mount()`: Setup before rendering (state, subscriptions)
  - `render_()`: Build the UI components
  - `responsive_UI()`: Handle window sizing and responsive layouts
</Accordion>

<Accordion title="Application Startup">
  ```python
  if __name__ == '__main__':
      appctxt = ApplicationContext()
      window = MyApp()
      if not app_is_frozen():
          window._init_hot_reload_system(__file__)
      window.show()
      exec_func = getattr(appctxt.app, 'exec', appctxt.app.exec_)
      sys.exit(exec_func())
  ```
  
  - Creates the application context
  - Instantiates the main window
  - Enables hot reloading in development mode
  - Shows the window and starts the event loop
</Accordion>

## Run Your Application

Start your application in development mode:

```bash
ppg start
```

You should see:

```
🔥 Hot Reload: Listening for changes in main.py
```

And a window displaying "Hello World!" will appear.

<Info>
  **Development Mode**: `ppg start` runs your app from source with hot reloading enabled. Changes to your code will be reflected instantly without restarting.
</Info>

## Experience Hot Reloading

Let's make a live change to see hot reloading in action:

<Steps>
  <Step title="Keep the App Running">
    Leave your application window open.
  </Step>
  
  <Step title="Edit main.py">
    Open `src/main/python/main.py` and modify the `render_()` method:
    
    ```python
    def render_(self):
        label = QLabel('Hello from PPG! 🚀', parent=self)
        label.setStyleSheet("""
            font-size: 24px;
            font-weight: bold;
            color: #2fe784;
            padding: 20px;
        """)
    ```
  </Step>
  
  <Step title="Save the File">
    Save `main.py`. You'll see console output:
    
    ```
    🔄 Hot Reload: Detected change in 'main.py'
    ⚡️ Hot Reload: Processing changes... (Reload #1)
    ✨ Hot Reload: Done. UI will update shortly.
    ```
  </Step>
  
  <Step title="See Instant Updates">
    Your application window automatically updates with the new styled label—no restart needed!
  </Step>
</Steps>

<Warning>
  **Hot Reloading in Production**: Remove the `@hot_reloading` decorator and `_init_hot_reload_system()` call before deploying to production. Hot reloading is a development-only feature.
</Warning>

## Build Configuration

The `src/build/settings/base.json` file contains your app's configuration:

```json base.json
{
    "app_name": "MyApp",
    "version": "1.0.0",
    "author": "John Doe",
    "binding": "PySide6",
    "main_module": "src/main/python/main.py",
    "hidden_imports": [
        "__future__"
    ]
}
```

- **app_name**: Your application's display name
- **version**: Semantic version (major.minor.patch)
- **binding**: Qt binding (PySide6, PyQt6, PySide2, or PyQt5)
- **main_module**: Entry point for your application
- **hidden_imports**: Modules to include when freezing your app

## CLI Commands Reference

PPG provides a complete set of CLI commands for your development workflow:

| Command | Description |
|---------|-------------|
| `ppg init` | Initialize a new project with interactive setup |
| `ppg start` | Run your app from source in development mode |
| `ppg create` | Generate new components or views from templates |
| `ppg freeze` | Compile your app into a standalone executable |
| `ppg installer` | Create a platform-specific installer (.exe, .dmg, etc.) |
| `ppg test` | Run your project's unit tests |
| `ppg clean` | Remove build artifacts and temporary files |
| `ppg version` | Display the current PPG version |

## Add a New Component

PPG can generate component scaffolding for you:

```bash
ppg create component
```

Follow the prompts:

```
Component name: UserProfile
Inherit from: QWidget
```

PPG creates `src/main/python/components/UserProfile.py`:

```python UserProfile.py
from ppg_runtime.application_context import Pydux, PPGLifeCycle, init_lifecycle
from PySide6.QtWidgets import QWidget

@init_lifecycle
class UserProfile(QWidget, PPGLifeCycle, Pydux):

    def component_will_mount(self):
        self.subscribe_to_store(self)

    def render_(self):
        # Render the UI here
        pass
```

You can now import and use this component in your application.
